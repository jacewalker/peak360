  <action>
Create `src/components/ui/ConfirmDeleteModal.tsx` as a 'use client' component.

Props interface:
```typescript
interface ConfirmDeleteModalProps {
  isOpen: boolean;
  itemCount: number;
  itemLabel: string;        // "assessment" or "client" — pluralized automatically
  onConfirm: () => void;    // Called after password verified successfully
  onCancel: () => void;
}
```

Implementation:
- Render nothing if `!isOpen`
- Overlay: fixed inset-0, bg-black/50, z-50, flex items-center justify-center, onClick onCancel
- Modal card: bg-white rounded-xl p-6 max-w-md w-full mx-4, onClick stopPropagation
- Header: red warning icon (SVG exclamation triangle) + "Confirm Deletion" in text-lg font-bold text-navy
- Body text: "This will permanently delete {itemCount} {itemLabel}(s). This action cannot be undone."
- Password input: type="password", placeholder="Enter your password to confirm", full width, standard border/focus styling matching the project (border-border, focus:border-gold/50, focus:ring-2 focus:ring-gold/10)
- Error state: red text below input showing error message
- Loading state: disable button, show spinner text
- Two buttons row (flex justify-end gap-3):
  - Cancel: text-sm text-muted hover:text-foreground border border-border rounded-lg px-4 py-2
  - Delete: text-sm text-white bg-red-600 hover:bg-red-700 rounded-lg px-4 py-2 font-medium, disabled when password empty or loading

On submit (form onSubmit or button click):
1. POST to `/api/auth/verify-password` with `{ password }`
2. If 401: set error "Incorrect password", clear password field
3. If success: call `onConfirm()`, reset internal state
4. If network error: set error "Something went wrong"

Internal state: password string, error string, loading boolean. Reset all when isOpen changes to true.
Auto-focus the password input when modal opens (useEffect with ref).
Handle Escape key to call onCancel.
