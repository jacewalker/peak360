# Quick 260525-fj2 — Functional Client login (magic-link default + optional password)

## Decisions (from user)
- Magic link is the **default** client sign-in.
- Optionally, a client may **set a password** and sign in with email + password.
- Keep it simple; reuse Better Auth methods already configured.

## Current state
- `src/app/login/page.tsx`: segmented `mode` = 'coach' | 'client'. Coach tab = email+password (`authClient.signIn.email`). **Client tab is a "Coming soon" placeholder** (~L144-154).
- `src/lib/auth.ts`: `emailAndPassword.disableSignUp: true` (line 19) — but the **magicLink plugin has NO disableSignUp**, so a public magic-link request could create accounts. `magicLink` already sends the branded email. `sendResetPassword` (branded) exists.
- `authClient` (auth-client.ts) has `magicLinkClient()` → `authClient.signIn.magicLink` is available. Better Auth core exposes `authClient.requestPasswordReset({ email, redirectTo })` and `authClient.resetPassword({ newPassword, token })`. `/reset-password` page already handles setting the new password from the token.

## Task 1 — Restrict magicLink to sign-in only

**File:** `src/lib/auth.ts`
- Add `disableSignUp: true,` to the `magicLink({ ... })` plugin options (alongside `expiresIn: 300`). This prevents a public magic-link request from minting a new account; the invite (`/api/invitations`) and client-login (`/api/client-login`) flows create the user via `auth.api.createUser` first, so they are unaffected.

**Verify:** `npx tsc --noEmit` clean.

## Task 2 — Make the Client tab functional

**File:** `src/app/login/page.tsx`

Replace the `mode === 'client'` placeholder block with a working flow. Add state:
`const [clientMode, setClientMode] = useState<'magic' | 'password'>('magic');`
`const [info, setInfo] = useState('');` (success/info messages). Reuse `email`, `password`, `error`, `loading`, `inputClasses`, the error block markup, and `MonoEyebrow`.

Handlers (all set loading, clear error/info, try/catch with a network-failure fallback message):
- `handleMagicLink` → `await authClient.signIn.magicLink({ email, callbackURL: '/portal' })`. On completion show an **info** message regardless of account existence (privacy): "If an account exists for that email, a secure sign-in link is on its way. Check your inbox." Do not navigate.
- `handleClientPassword` → `await authClient.signIn.email({ email, password, callbackURL: '/portal' })`. On `res.error` → "Invalid email or password." else `router.push('/portal'); router.refresh();`.
- `handleSetPassword` → require `email`; `await authClient.requestPasswordReset({ email, redirectTo: '/reset-password' })`. Show info: "If an account exists for that email, we've sent a link to set your password."

Client-tab UI (magic link = primary/default):
- A short intro: "Sign in to view your assessments."
- **Default (`clientMode === 'magic'`):** email input + a **primary gold button** "Email me a login link" (calls `handleMagicLink`; disabled when no email/loading). Below it, a secondary text link: "Prefer a password? Sign in with a password" → `setClientMode('password')`.
- **Password (`clientMode === 'password'`):** email + password inputs + a primary "Sign in" button (`handleClientPassword`). A secondary link "Email me a login link instead" → `setClientMode('magic')`. And a text link "First time, or forgot it? Set / reset your password" → `handleSetPassword`.
- Render `info` (gold/neutral) and `error` (danger) messages using the existing styling language.
- Keep the Coach tab unchanged.

**Verify:** `npx tsc --noEmit` clean; `npx eslint src/app/login/page.tsx` clean. Confirm `authClient.signIn.magicLink` and `authClient.requestPasswordReset` exist on the typed client (adjust to `forgetPassword` only if `requestPasswordReset` is not typed in the installed better-auth).
