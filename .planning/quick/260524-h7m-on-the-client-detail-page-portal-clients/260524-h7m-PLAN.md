# Quick 260524-h7m â€” Start assessment from the client detail page

## Objective

Add a "Start assessment" button to the client detail page hero
(`src/app/portal/clients/[name]/page.tsx`) that creates a new assessment
already assigned to that client, seeds Section 1, and opens it. The client is
known from the route param, so NO client picker is shown â€” this is the
auto-assigned shortcut.

## Context / reuse

The dashboard already has the canonical helper (`handleCreateForClient(name)`
in `src/app/portal/page.tsx`):

```ts
const res = await fetch('/api/assessments', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ clientName: name }),
});
const { data } = await res.json();
await fetch(`/api/assessments/${data.id}/sections/1`, {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ data: { clientName: name } }),
});
router.push(`/portal/assessment/${data.id}/section/1`);
```

The client detail page is a `'use client'` component and already derives
`clientName` from the route (`decodeURIComponent(params.name)`). It does NOT
currently import `useRouter`.
