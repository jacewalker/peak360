# Quick 260524-jk6 — Clients page: show which client is selected

## Bug

On `/portal/clients`, ticking a client's checkbox reveals the "Delete N selected"
button but gives **no visual feedback on the card itself** — the card border
stays `border-line` whether selected or not, and the only indicator is a small
16px native checkbox that's easy to miss on the dark card. Users can't tell which
client(s) they've selected.

## Current code (for reference)

`src/app/portal/clients/page.tsx`, card render (~L284–335):
- The card is a `<Link>` with a FIXED class:
  `className="block bg-bg-3 rounded-xl border border-line p-6 hover:border-gold-brand/40 transition-colors"`
- Selection state: `selectedNames.has(c.name)` (a `Set<string>`); toggled by `toggleSelectOne(c.name)`.
- Checkbox (~L292): `<input type="checkbox" className="w-4 h-4 rounded border-line accent-gold-brand" checked={selectedNames.has(c.name)} ... />` inside a `<div onClick={(e) => e.preventDefault()}>` wrapper.

## Task 1 — Add a selected-state style to the card

**File:** `src/app/portal/clients/page.tsx`

**Action:**
1. Make the card `<Link>` className conditional on `selectedNames.has(c.name)`:
   - **Selected:** clear, on-brand emphasis — gold border + ring + subtle tint, e.g.
     `border-gold-brand ring-1 ring-gold-brand/40 bg-gold-brand/5`
   - **Unselected:** keep current — `border-line hover:border-gold-brand/40`
   - Keep the shared classes (`block bg-bg-3 rounded-xl border p-6 transition-colors`). Note: move `border` into the shared part and let the colour be conditional so there's no double `border` utility. Example:
     ```tsx
     className={`block rounded-xl border p-6 transition-colors ${
       selectedNames.has(c.name)
         ? 'border-gold-brand ring-1 ring-gold-brand/40 bg-gold-brand/5'
         : 'bg-bg-3 border-line hover:border-gold-brand/40'
     }`}
     ```
     (Selected state may override `bg-bg-3` with the tint, or keep `bg-bg-3` and add the tint — pick whichever reads cleanly; the tint must stay subtle and legible in the dark theme.)
2. Make the checkbox more visible: bump it to `w-5 h-5` and keep `accent-gold-brand` (and a visible border) so the tick is obvious. Do not change selection logic, the `onClick preventDefault` wrapper, or `aria-label`.
3. Do not touch the list/grid, search, coach filter, select-all, or delete logic.
