## Spacing Scale

Declared values (must be multiples of 4 — matches existing usage in `Sidebar.tsx`, `portal/admin/page.tsx`, `login/page.tsx`):

| Token | Tailwind | Value | Usage in this phase |
|-------|----------|-------|----------------------|
| xs | `1` | 4px | Icon-to-text gap inside buttons; pill chip padding |
| sm | `2` | 8px | Form-field internal vertical rhythm; toggle option padding |
| md | `3`–`4` | 12–16px | Default padding inside list rows, table cells, glass-card field gaps |
| lg | `6` | 24px | Card body padding; dashboard section vertical spacing; glass-card padding (`p-6` to `p-8`) |
| xl | `8` | 32px | Page heading bottom margin; group-section gap on coach-grouped dashboard |
| 2xl | `12` | 48px | Hero header vertical padding (admin pages); top padding of `/login` content shell |
| 3xl | `14`–`16` | 56–64px | Auth shell vertical centering (already `min-h-screen flex items-center`) |

**Exceptions:**
- Sidebar nav items use `px-3 py-2.5` (10px Y) — already established in `Sidebar.tsx`; keep verbatim for new role-conditional items so they visually match Dashboard/Assessments/Clients/Admin.
- Glass card on `/login` and `/reset-password`: `p-8` (32px) interior padding — preserves the existing visual; new "Forgot password?" / magic-link CTAs sit inside the same card, NOT in a new container.
- "Forgot password?" link spacing: `mt-3` (12px) below password field — tighter than primary CTA spacing to read as a tertiary action.
- Magic-link secondary CTA spacing: separated from primary submit by an `mt-4` divider row (16px) with a `text-white/30 text-[10px]` "OR" hairline label.

## Color

60/30/10 contract. Two distinct contexts coexist: the **unauthed glassmorphic shell** (`/login`, `/reset-password`) and the **portal/admin shell** (everything under `/portal`). Both share the same accent rules.

### Unauthed shell (`/login`, `/reset-password`)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `linear-gradient(135deg, #0f2440 → #1a365d → #2d5986)` (`--color-navy-dark` → `--color-navy` → `--color-navy-light`) | Full-screen background gradient (already on `/login`); apply identical gradient to `/reset-password` per D-24 |
| Secondary (30%) | `bg-white/[0.07]` over `border-white/10` with `backdrop-blur-xl` | Glass card containing the form; field backgrounds at `bg-white/[0.06]`; "OR" hairline divider between primary and magic-link CTAs |
| Accent (10%) | `--color-gold` (`#F5A623`) | Primary CTA fill ("Sign In", "Send reset link", "Send sign-in link"); inline link emphasis ("Forgot password?", "Email me a sign-in link"); 12px gold underline accent under logo |
| Destructive | `text-red-400` (Tailwind default) | Inline error rows under forms (already established at `login/page.tsx:172-179`) |

### Portal / admin shell (`/portal/*`)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-background` `#f8fafc` | Page background for all `/portal` and `/portal/admin/*` routes |
| Secondary (30%) | `--color-surface` `#ffffff` over `--color-border` `#e2e8f0` | Card surfaces (assessment list rows, user table, invitation list/form panels, group-section containers); `--color-navy-dark` for the fixed `Sidebar` shell (already established) |
| Accent (10%) | `--color-gold` `#F5A623` | Primary CTAs only (see reserved list below); active sidebar item background `bg-gold/15` and active dot indicator |
| Destructive | `--color-rating-poor` `#ef4444` | "Demote admin" disabled tooltip text; toast for last-admin guard failure; PDF download error banner on client report |

### Accent reserved-for list (BINDING — do NOT use gold elsewhere)

Gold (`--color-gold`) is reserved EXCLUSIVELY for these elements in this phase:

1. The single primary submit button on each form: `Sign In`, `Send reset link`, `Set new password`, `Send invitation`, `Save role` (in users page row).
2. Active sidebar nav item (background tint + dot indicator) — already established; new conditional items inherit identically.
3. The `Download PDF` button on the client report page (`/portal/assessment/[id]/report`).
4. Inline link emphasis on `/login` for the secondary CTA labels ("Forgot password?", "Email me a sign-in link").
5. The 2px gold accent line under the Peak360 logo on `/login` and `/reset-password`.
6. The Coach selector tag/eyebrow on the admin grouped dashboard ("My clients (you)" header gets a gold left border `border-l-4 border-gold`; other coach groups get a navy left border `border-l-4 border-navy`).

Forbidden gold uses in this phase:
- Not on table row hover states (use `bg-surface-alt` `#f1f5f9` instead).
- Not on secondary buttons (use `bg-surface` with `border-border` and `text-navy` text).
- Not on the role pill in the user table (use neutral pills — see Pill spec below).
- Not on info banners or empty-state cards.

## Interaction & State Contract

### Loading states
- Auth submit buttons show inline spinner replacing label text (existing pattern at `login/page.tsx:186-189` — KEEP).
- Sidebar role-conditional items: render `Dashboard` immediately; mount role-specific items only after `useSession()` resolves. **Privileged items (`Admin`) MUST NOT flash for non-admin sessions** (acceptance criterion from CONTEXT D-12).
- Admin user table: skeleton rows while loading (`bg-surface-alt` shimmer is overkill — just render 5 rows of `bg-surface-alt h-12 rounded`).
- Toast appears within 200ms of action confirmation/failure.

### Error states
- All API errors surface as either inline form errors (for synchronous submit failures) or toasts (for async row mutations like role change).
- Network errors get the same generic copy regardless of cause: `"We couldn't reach the server. Check your connection and try again."`
- 403 errors on the `/portal/admin/*` pages redirect to `/portal` with a toast: `"You don't have access to that page."`

### Empty states
All empty states have an icon + heading + body + (optional) CTA, vertically centered in the available space, max-width `max-w-sm`. Icons are Heroicons-style outline strokes at `w-12 h-12 text-muted` — pick a contextually relevant glyph (clipboard for assessments, users-group for admin user list, paper-airplane for invitations).

### Focus / a11y
- All interactive elements MUST have a visible focus ring. Inputs already have the gold focus ring globally. Buttons get `focus-visible:ring-2 focus-visible:ring-gold focus-visible:ring-offset-2`.
- Mode toggle: arrow keys move between options; `aria-pressed` on each option.
- Toast: `role="status"` for success, `role="alert"` for error.
- Sidebar role-specific items: `aria-current="page"` on the active item (already in place).
- The "Forgot password?" link is a real `<button type="button">`, not a `<Link>` — it triggers the reset-request POST inline.

### Responsive
- All new pages MUST work at 360px width minimum.
- Admin user table and invitations table: stack into card list at `< 768px` (md breakpoint).
- Coach-grouped dashboard: groups stack vertically at all widths (no side-by-side layout).
- Sidebar already collapses to mobile drawer at `< lg` (1024px) — KEEP unchanged.
- Glass cards on `/login` and `/reset-password`: `max-w-sm` (existing) — works at 360px.
