<objective>
Wire Better Auth's password reset path end-to-end and lock down public signup. After this plan:

Purpose: Closes the "raw SQL or env-var reset" recovery gap that surfaced in prod on 2026-05-07 and removes the open-internet self-register-as-coach path documented in 07-SPEC.md req 2.

Output: Modified `src/lib/auth.ts` + new `src/app/reset-password/page.tsx`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous user → Better Auth signup endpoint | Open-internet POST surface; D-01 closes this. |
| anonymous user → /reset-password page | Untrusted token traversal; Better Auth verifies + expires the token server-side. |
| anonymous user → /api/auth/forget-password (Better Auth) | Untrusted email enumeration surface (Better Auth handles user-existence non-disclosure). |

## STRIDE Threat Register

| Threat ID  | Category               | Component                              | Disposition | Mitigation Plan |
|------------|------------------------|----------------------------------------|-------------|-----------------|
| T-07-01    | Spoofing / Elevation   | Better Auth signup endpoint            | mitigate    | `disableSignUp: true` in `auth.ts` (D-01). Better Auth returns its standard "Sign up is disabled" error for any anonymous POST. Defence-in-depth: the "Create one" link is removed from /login in plan 07-04. |
| T-07-02    | Tampering              | Reset-password token in URL            | mitigate    | Better Auth signs and time-bounds reset tokens server-side (1-hour expiry per email body). Page surfaces "expired link" error on `res.error`; never silently succeeds. |
| T-07-03    | Repudiation            | Self-service password reset            | accept      | Audit logging on the reset itself is out of scope for this phase. Email delivery + Better Auth's session invalidation provide acceptable evidence trail. (Audit logging is reserved for admin role-changes in plan 07-07.) |
| T-07-04    | Information Disclosure | sendResetPassword email body           | mitigate    | Email body contains only the generic reset URL (no password, no PII beyond the reset link). Subject line is non-revealing ("Reset your Peak360 password"). |
| T-07-05    | DoS                    | /api/auth/forget-password rate limit   | accept      | Better Auth's built-in rate limiting on forget-password endpoint is sufficient for solo-operator deployment. Rate-limit hardening is not in 07-SPEC scope. |
| T-07-06    | Spoofing               | Misconfiguration regression (signup re-enabled) | mitigate | Plan 07-11 contributes a regression test asserting `disableSignUp: true` is present in auth.ts. |
</threat_model>
