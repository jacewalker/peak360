<objective>
Close two UX/security blockers in the portal shell.

**BL-03 — VERIFICATION.md row 1 ("Logout properly invalidates the session cookie"):**
src/components/layout/Sidebar.tsx line 190 calls fetch('/api/auth/logout', { method: 'POST' }). Better Auth exposes sign-out at /api/auth/sign-out, NOT /logout. The fetch returns 404, but the .finally() redirect to /login happens regardless — meaning the user is sent back to the login page WITH a still-valid session cookie. Pressing browser back re-enters the portal as the previous user. Fix: replace the fetch with authClient.signOut() from @/lib/auth-client, which invokes Better Auth's sign-out endpoint correctly and clears the cookie via Set-Cookie headers.

**BL-04 — VERIFICATION.md row 3 ("Clients do not see the '+ New Assessment' creation CTA on the dashboard"):**
src/app/portal/page.tsx lines 168-173 render a creation button unconditionally. REQ-7.12's acceptance criterion: "logged in as a client with zero assessments, the dashboard shows the client-specific empty message and no creation CTA." The empty-state text in the Recent Assessments card is correctly gated, but the primary header CTA is not. Fix: wrap the button in a strict-positive role guard so it is absent from the DOM for clients but unchanged for coaches/admins.

The existing tests/security/sidebar-role-flash.test.tsx regression test currently has no assertion about the logout button's wiring; we add one so a future regression to the broken endpoint trips the test.

Output: 2 source files modified + 1 test file extended.
