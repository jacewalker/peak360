Output: `markers` table live in Postgres; query layer + merge helpers + pillar short-circuit + new AuditAction members all type-checking and unit-tested.

MarkerRow (src/lib/markers/queries.ts):
  interface MarkerRow {
    testKey: string;
    label: string;
    section: number;          // 1..10
    dataKey: string;          // camelCase, written to sectionData JSON
    pillar: PillarKey;        // one of PILLAR_KEYS
    category: string;
    subcategory: string | null;
    fallbackUnit: string | null;
    hasNorms: boolean;
    aiAliases: string[] | null;
    severityWeight: number | null;
    createdBy: string;
    createdAt: number;        // epoch ms
    updatedBy: string;
    updatedAt: number;        // epoch ms
  }

RegistryMarker (src/lib/markers/registry.ts) extends the shape of REPORT_MARKERS[]'s MarkerDef:
  type RegistryMarker = MarkerDef & {
    pillar?: PillarKey;       // present only on DB rows
    source: 'seed' | 'db';
    aiAliases?: string[];
    severityWeight?: number | null;
    createdAt?: number;
    updatedAt?: number;
  }

getReportMarkers(): Promise<RegistryMarker[]>
  - seed first (REPORT_MARKERS order, skipping any testKeys also in DB), then DB rows in createdAt ascending
  - DB wins on testKey conflict (per D-01)

getFieldMappings(): Promise<Record<string, string>>
  - { ...hardcoded fieldMappings, ...flatten(dbRows[].aiAliases -> dataKey, lowercased+trimmed) }
  - DB wins on alias collision (per D-04)

AuditAction additions (src/lib/audit.ts): 'marker.create' | 'marker.update' | 'marker.delete'

PILLAR_KEYS (src/lib/pillars/keys.ts or src/types/pillars.ts): 'cardiometabolic' | 'bodyComposition' | 'strength' | 'balance' | 'vo2'
REPORT_MARKERS (src/lib/report-markers.ts): MarkerDef[] - seed remains the source of ~98 seeded markers (D-03)
