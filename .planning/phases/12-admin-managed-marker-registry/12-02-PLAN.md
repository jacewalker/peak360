POST /api/admin/markers request body:
  {
    testKey: string;          // ^[a-z][a-z0-9_]*$
    label: string;            // non-empty
    section: number;          // integer 1..10
    dataKey: string;          // ^[a-z][a-zA-Z0-9]*$
    pillar: 'cardiometabolic' | 'bodyComposition' | 'strength' | 'balance' | 'vo2';
    category: string;
    subcategory?: string | null;
    fallbackUnit?: string | null;
    hasNorms: boolean;
    aiAliases?: string[] | null;     // optional
    severityWeight?: number | null;
    initialTiers?: {                  // required when hasNorms === true (unisex initial range, D-05)
      poor: { min: number|null; max: number|null };
      cautious: { min: number|null; max: number|null };
      normal: { min: number|null; max: number|null };
      great: { min: number|null; max: number|null };
      elite: { min: number|null; max: number|null };
    };
    initialUnit?: string | null;      // unit for initial range row (optional)
  }

Success: { success: true, data: { marker: MarkerRow } }
Failures:
  - 400 validation error (regex fail, enum fail, missing required, invalid initialTiers)
  - 401 not signed in / 403 not admin (via requireAdmin)
  - 409 { success: false, error: 'testKey conflicts with a seeded marker' } when testKey matches any REPORT_MARKERS entry
  - 409 { success: false, error: 'testKey already in use' } when DB row exists

PUT /api/admin/markers/[testKey] body: same shape minus testKey, PLUS `updatedAt: number` for optimistic concurrency.
  - 400 if body.dataKey differs from existing.dataKey
  - 409 if existing.updatedAt > body.updatedAt
  - On success returns { success: true, data: { marker } } and emits 'marker.update' audit

DELETE /api/admin/markers/[testKey]:
  - Cascade-deletes marker, marker_content, normative_ranges via queries.deleteMarker()
  - Emits 'marker.delete' audit with metadata { deletedContent, deletedRanges }
  - Returns { success: true, data: { deletedMarker, deletedContent, deletedRanges } }

GET /api/markers (any authenticated role):
  - Returns { success: true, data: { markers: RegistryMarker[] } } via getReportMarkers()
  - 401 if not signed in
