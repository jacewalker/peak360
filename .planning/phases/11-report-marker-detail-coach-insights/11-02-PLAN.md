## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API (admin) | Untrusted PUT body crosses into a write on global content |
| client→API (read) | Any authenticated session reads global content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04 | Elevation of Privilege | PUT/GET on /api/admin/marker-content/* | mitigate | requireAdmin() gate first in every handler; returns 403 for non-admin, 401 for unauthenticated (clone of normative routes) |
| T-11-05 | Tampering | concurrent admin edits overwriting each other | mitigate | 409 optimistic-concurrency check on `updatedAt` before upsert (clone normative semantics) |
| T-11-06 | Repudiation | admin content edits untracked | mitigate | logAuditEvent('marker_content.update') with userId, resourceId=testKey, ipAddress, userAgent on every PUT |
| T-11-07 | Information Disclosure | client route over-exposing data | accept | marker_content is non-PII global clinical content; requireSession ensures only authenticated users read (D-07/D-12) |
</threat_model>
