<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| seed → DB | Static seed content inserted into a shared global table; no user input |
| schema migration | DDL executed at app startup against the live DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | seed overwriting admin edits | mitigate | Idempotent insert-if-absent (ON CONFLICT DO NOTHING / INSERT OR IGNORE); never DO UPDATE — admin content is never clobbered (D-09) |
| T-11-02 | Denial of Service | runMigrations seed loop (~98 inserts) at startup | accept | One-time idempotent cost; subsequent boots no-op on conflict; mirrors Phase 8 pillar seed already in production |
| T-11-03 | Information Disclosure | marker_content holds no PII | accept | Global clinical content only, no client data |
</threat_model>
