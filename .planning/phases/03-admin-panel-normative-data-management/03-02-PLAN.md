<objective>
Implement normative range versioning so each assessment locks in the normative ranges active at creation time, and the report uses those versioned ranges.

Purpose: Ensures existing assessments retain their original rating context even after admins edit ranges. This is a core data integrity guarantee -- without versioning, changing a threshold retroactively alters past assessment reports.

Output: A versioning module (create/reuse snapshots), integration into assessment creation API, and updated Section 11 report to use versioned ranges when available.
</objective>

1. Create `src/lib/normative/versioning.ts`:

Import `crypto` from Node.js (for SHA-256), `db` from `@/lib/db`, uuid from `uuid`, schema detection pattern from db-ranges.ts.

**mergeDbWithHardcoded():** Async function that:
- Calls `getAllDbRanges()` from db-ranges.ts to get all DB overrides
- Imports `normativeData` from `./data` (hardcoded)
- Builds a complete merged object: start with a deep clone of normativeData, then overlay any DB range rows on top
- For each DB row, the key structure is: `{ [testKey]: { unit, note, tiers, gender?, ageGroup?, severityWeight? } }`
- The merged object format: `Record<string, { unit?, note?, severityWeight?, variants: Array<{ gender: string|null, ageGroup: string|null, tiers: TierRanges }> }>`
- Returns the merged object as a plain JSON-serializable value

**computeContentHash(mergedJson: string):** Synchronous function. Takes the JSON.stringify'd merged data (with keys sorted for determinism -- use `JSON.stringify(obj, Object.keys(obj).sort())`). Returns SHA-256 hex digest using `crypto.createHash('sha256').update(mergedJson).digest('hex')`.

**createOrReuseVersion():** Async function that:
- Calls `mergeDbWithHardcoded()` to get the current merged state
- Serializes to JSON with sorted keys for deterministic hashing
- Computes content hash via `computeContentHash()`
- Queries `normativeVersions` table for existing row with same `contentHash`
- If found: return the existing version id (dedup -- no new row created)
- If not found: generate UUID via `crypto.randomUUID()`, INSERT new row with id, rangesJson, contentHash, createdAt = `new Date().toISOString()`
- Return the version id (string)

**getVersionSnapshot(versionId: string):** Async function that:
- Queries `normativeVersions` table by id
- Returns parsed `rangesJson` as `Record<string, unknown>` or null if not found

**GET handler:** Queries all rows from normativeVersions ordered by createdAt DESC. Returns `{ success: true, data: versions }` where each version includes id, contentHash, createdAt (but NOT the full rangesJson to keep response small). Follow existing API pattern: `NextResponse.json({ success: true, data: [...] })`.
