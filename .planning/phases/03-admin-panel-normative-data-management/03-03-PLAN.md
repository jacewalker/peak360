Create `src/app/admin/normative/page.tsx` as a client component (`'use client'`):

**State:**
- `dbOverrideKeys: Set<string>` -- testKeys that have DB overrides (loaded from API)
- `loading: boolean` -- true while fetching
- `error: string | null` -- error message
- `searchQuery: string` -- text filter
- `filterMode: 'all' | 'db_overrides' | 'hardcoded'` -- dropdown filter

**On mount useEffect:**
- Fetch GET `/api/admin/normative`
- On success: extract `overrideKeys` array, build `new Set(overrideKeys)`, set `dbOverrideKeys`, set `loading = false`
- On error: set `error = 'Could not load normative data. Refresh the page to try again.'`

**Import REPORT_MARKERS** from `@/lib/report-markers`.

**Derive categories:** `[...new Set(REPORT_MARKERS.map(m => m.category))]` preserving insertion order.

**Filter logic:**
- Start with all `REPORT_MARKERS`
- If `searchQuery` is non-empty, filter where `m.label.toLowerCase().includes(searchQuery.toLowerCase())`
- If `filterMode === 'db_overrides'`, filter where `dbOverrideKeys.has(m.testKey)`
- If `filterMode === 'hardcoded'`, filter where `m.hasNorms && !dbOverrideKeys.has(m.testKey)`

**Render structure (following UI-SPEC exactly):**

```
<div className="max-w-6xl mx-auto p-6">
  <!-- Page title -->
  <h1 className="text-2xl font-bold text-navy mb-6">Normative Range Management</h1>

  <!-- Search/filter bar: sticky top bar -->
  <div className="sticky top-0 z-10 bg-background pb-4 flex gap-3">
    <input
      type="text"
      placeholder="Search markers..."
      value={searchQuery}
      onChange={e => setSearchQuery(e.target.value)}
      className="flex-1 px-3 py-2 border border-border rounded-lg text-sm focus:border-gold focus:ring-2 focus:ring-gold/25 outline-none"
    />
    <select
      value={filterMode}
      onChange={e => setFilterMode(e.target.value as ...)}
      className="px-3 py-2 border border-border rounded-lg text-sm"
    >
      <option value="all">All Markers</option>
      <option value="db_overrides">DB Overrides Only</option>
      <option value="hardcoded">Hardcoded Only</option>
    </select>
  </div>

  <!-- Loading state: 6 category blocks with 4 shimmer rows -->
  {loading && <LoadingSkeleton />}

  <!-- Error state -->
  {error && <div className="p-4 bg-red-50 text-red-600 rounded-lg text-sm">{error}</div>}

  <!-- Category sections -->
  {!loading && !error && categories.map(cat => {
    const markers = filteredMarkers.filter(m => m.category === cat);
    if (markers.length === 0) return null;
    const dbCount = markers.filter(m => dbOverrideKeys.has(m.testKey)).length;
    return (
      <section key={cat} className="mb-8">
        <div className="flex items-center gap-3 border-b border-border pb-2 mb-3">
          <h2 className="text-xl font-bold text-navy">{cat}</h2>
          <span className="text-xs text-muted">{markers.length} markers</span>
          {dbCount > 0 && <span className="text-xs text-muted">{dbCount} with DB overrides</span>}
        </div>
        <div className="grid gap-2">
          {markers.map(m => (
            <a key={m.testKey} href={`/admin/normative/${m.testKey}`}
               className="flex justify-between items-center p-3 bg-surface border border-border rounded-lg hover:border-gold transition-colors">
              <span className="text-sm font-bold text-navy">{m.label}</span>
              <StatusPill testKey={m.testKey} hasNorms={m.hasNorms} isDbOverride={dbOverrideKeys.has(m.testKey)} />
            </a>
          ))}
        </div>
      </section>
    );
  })}

  <!-- Empty state when filters produce no results -->
  {!loading && !error && filteredMarkers.length === 0 && (
    <div className="text-center py-12">
      <h3 className="text-xl font-bold text-navy mb-2">No DB Overrides Yet</h3>
      <p className="text-sm text-muted">All markers are using hardcoded defaults. Select a marker to customize its ranges.</p>
    </div>
  )}

  <!-- Version info footer -->
  <p className="text-xs text-muted mt-8">Ranges are versioned automatically. Existing assessments keep the ranges they were created with.</p>
</div>
```
