**GET handler:**
- Extract `marker` (testKey) from params: `const { marker } = await params;`
- Find marker metadata from REPORT_MARKERS: `REPORT_MARKERS.find(m => m.testKey === marker)`
- If marker not found in REPORT_MARKERS, return 404: `{ success: false, error: 'Marker not found' }`
- Call `getDbRangesByTestKey(marker)` to get all DB override variants
- Also call `getStandards(marker, null, null)` to get hardcoded defaults (for display when no DB override)
- For gendered markers, also call `getStandards(marker, null, 'male')` and `getStandards(marker, null, 'female')`
- Return: `{ success: true, data: { marker: markerDef, dbOverrides: dbRows, hardcodedDefaults: { unisex?, male?, female? } } }`

**PUT handler:**
- Extract `marker` from params
- Parse request body: `{ variants: Array<{ gender: string|null, ageGroup: string|null, tiers: TierRanges, unit?: string, note?: string, severityWeight?: number }> }`
- Find marker in REPORT_MARKERS to get category
- Validate each variant's tiers: for each tier, `min < max`. For adjacent tiers, `poor.max === cautious.min`, `cautious.max === normal.min`, etc. Return 400 with specific error if validation fails.
- For each variant, call `upsertDbRange({ testKey: marker, category: markerDef.category, gender: variant.gender, ageGroup: variant.ageGroup, unit: variant.unit, note: variant.note, tiers: variant.tiers, severityWeight: variant.severityWeight })`
- Return: `{ success: true, data: { updated: variants.length } }`
- Include optimistic locking: accept `updatedAt` in body, compare with current DB row's updatedAt. If mismatch, return 409: `{ success: false, error: 'This marker was updated by another admin. Reload to see their changes before saving.' }`

**DELETE handler:**
- Extract `marker` from params
- Parse optional body: `{ gender?: string, ageGroup?: string }` -- if provided, delete specific variant; if not, delete ALL variants for this testKey
- Call `deleteDbRange(marker, gender, ageGroup)`
- Return: `{ success: true, data: { deleted: count } }`

Create `src/app/admin/normative/[marker]/page.tsx` as a client component (`'use client'`):

**Props:** Page receives `params: Promise<{ marker: string }>` (Next.js 16 dynamic route params).

**State:**
- `markerDef: MarkerDef | null` -- metadata from REPORT_MARKERS (looked up client-side)
- `dbOverrides: NormativeRangeRow[]` -- existing DB rows for this marker
- `hardcodedDefaults: Record<string, Standards>` -- hardcoded values for comparison
- `editTiers: Record<string, TierRanges>` -- current form state keyed by variant key (e.g., "unisex", "male", "female", "male_18-25")
- `severityWeight: number` -- 0-10
- `isDirty: boolean` -- tracks unsaved changes
- `isSaving: boolean` -- loading state for save
- `saveSuccess: boolean` -- brief flash after save
- `error: string | null`
- `validationErrors: Record<string, string[]>` -- per-variant validation errors
- `activeGender: 'all' | 'male' | 'female'` -- gender tab
- `activeAgeGroup: string | null` -- age group dropdown
- `serverUpdatedAt: string | null` -- for optimistic locking

**On mount:**
- Resolve `marker` from params
- Find `markerDef` from `REPORT_MARKERS.find(m => m.testKey === marker)`
- Fetch GET `/api/admin/normative/${marker}`
- Populate `dbOverrides` and `hardcodedDefaults` from response
- Initialize `editTiers`: if DB overrides exist, use those tiers; otherwise, use hardcoded defaults
- Initialize `severityWeight` from first DB override row's severityWeight, or 0

**Variant detection:**
Determine marker type by examining hardcoded data structure or DB overrides:
- If only unisex data (no gender variants): show single tier form, no tabs
- If gendered (male/female variants): show gender tabs "All" / "Male" / "Female"
- If gendered + age-bucketed: show gender tabs + age group dropdown with options matching the hardcoded data's age groups

**handleTierChange(tier, field, value):**
- Update editTiers for current variant
- Set isDirty = true
- Run validation: check min < max for each tier, check no gaps between adjacent tiers
- Update validationErrors state

**Validation function validateTiers(tiers: TierRanges):**
- For each tier: if min >= max, add error: "Min must be less than max" for that tier
- For adjacent pairs (poor->cautious, cautious->normal, normal->great, great->elite): if prev.max !== next.min, add error: "Gap between tiers: {prev} max ({value}) must equal {next} min ({value})"
- Return array of error strings

**handleSave():**
- Set isSaving = true
- Build variants array from editTiers (include severityWeight on each)
- PUT to `/api/admin/normative/${marker}` with `{ variants, updatedAt: serverUpdatedAt }`
- On success (200): set isDirty = false, set saveSuccess = true, setTimeout to clear saveSuccess after 2s, update serverUpdatedAt
- On 409 (conflict): set error to the conflict message from response
- On other error: set error to "Failed to save range changes. Check your connection and try again."
- Set isSaving = false

**handleReset():**
- Show browser confirm dialog: "This will remove all custom ranges for {marker_name} and revert to hardcoded values. This cannot be undone."
- If confirmed: DELETE to `/api/admin/normative/${marker}`
- On success: reload editTiers from hardcoded defaults, set isDirty = false, clear dbOverrides
