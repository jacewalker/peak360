# Phase 8 — UI Design Contract

> Visual and interaction contract for the Peak Living five-pillar report refresh (portal + PDF mirror) and admin authoring surfaces. Generated by gsd-ui-researcher; verified by gsd-ui-checker.

This contract is constrained by upstream `08-CONTEXT.md` decisions D-01 → D-30 and the project-wide tokens already in `src/app/globals.css`. Where CONTEXT locks a value, this spec carries it forward verbatim. Where CONTEXT delegates to "Claude's discretion" (card chrome, micro-interactions, disclosure pattern), this spec prescribes a single answer so the planner and executor have no ambiguity.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no `components.json`; Tailwind v4 with `@theme inline` in `src/app/globals.css`) |
| Preset | not applicable |
| Component library | none — hand-rolled primitives consistent with existing `src/components/forms/*` and `src/components/sections/Section11.tsx` |
| Dialog primitive | new in this phase: `src/components/ui/Dialog.tsx` — hand-rolled, `role="dialog"`, `aria-modal="true"`, focus trap, `Esc` + backdrop close, `inert` background, `prefers-reduced-motion` respected. Bottom-sheet variant (mobile) animates from `translateY(100%)` to `0` over 200ms; centred-dialog variant (≥ md) fades + scales (95% → 100%) over 150ms. **No new npm dependency** — focus trap implemented inline with the established `useEffect` + `tabbable` walk pattern used elsewhere. (Planner may swap in Radix Dialog only if a separate `RESEARCH.md` task surfaces that the hand-rolled approach blocks delivery; this contract defaults to hand-rolled.) |
| Icon library | inline SVG (consistent with existing report components — no `lucide-react` / `heroicons` dependency). One new icon needed: chevron / arrow-right glyph for the "drill down" affordance on each pillar card. Inline SVG, 16px stroke width 2, `currentColor`. |
| Font | Inter (already loaded via `--font-sans` in `globals.css`); PDF reuses `src/lib/pdf/fonts.ts`. **Montserrat / Open Sans landing-page fonts are NOT used in the report.** |

## Spacing Scale

Tailwind's default 4px scale is the source of truth. Declared values used in this phase:

| Token | Value | Tailwind | Usage in this phase |
|-------|-------|----------|---------------------|
| xs | 4px | `1` | Icon-to-text gaps, traffic-light dot to label |
| sm | 8px | `2` | Compact spacing inside the score breakdown rows; bullet-list gaps in modal |
| md | 16px | `4` | Default padding inside pillar cards on mobile; gap between pillar cards on mobile |
| lg | 24px | `6` | Pillar card padding (desktop); gap between pillar cards on desktop; modal section spacing |
| xl | 32px | `8` | Page heading bottom margin; gap between hero copy and pillars grid |
| 2xl | 48px | `12` | Major section breaks (e.g., between PillarsGrid and the collapsed "Detailed marker results" disclosure) |
| 3xl | 64px | `16` | Page-level top/bottom padding on the report shell |

Exceptions:
- **Modal close button hit target**: 44 × 44px (touch target minimum). Visual icon stays at 16 × 16px; padding fills to 44px.
- **Bottom-sheet drag handle**: 36 × 4px pill, centred, 8px top margin (mobile only). Decorative — visible affordance for swipe-to-dismiss.
- **Pillar card on mobile**: minimum tappable height 96px so the entire card behaves as one large hit target (cards open the modal on tap).
- **PDF spacing**: PDF mirror uses the existing `src/lib/pdf/styles.ts` scale (different absolute pt values from CSS px). Do NOT introduce new PDF spacing tokens. The pillar grid uses existing `mb-md` / `mb-lg` style equivalents.

## Typography

Four sizes, two weights. Body line-height 1.5; heading line-height 1.2.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Modal prose ("What this pillar means", interpretation, prescription summary, page intro) |
| Label | 14px | 600 (semibold) | 1.4 | Traffic-light status label ("Strong" / "Needs improvement" / "Priority"), pillar card subtitle, modal section labels ("Score breakdown", "Recommended plan"), admin form labels |
| Heading | 20px | 600 (semibold) | 1.25 | Pillar card name; modal section headings inside the body; collapsed disclosure label ("Detailed marker results") |
| Display | 28px | 600 (semibold) | 1.15 | Page heading "The Peak Living Pillars"; modal pillar name in the modal header |

**Score number** (the `/100` figure): treated as a tabular numeric, **40px / weight 600 / line-height 1** on the pillar card; **48px / weight 600 / line-height 1** in the modal header. This is a deliberate exception to the four-role table — large numerals are a focal point per Kevin's brand direction and need their own scale. Use `font-variant-numeric: tabular-nums` so single-digit and three-digit scores align.

**Mobile downscaling**: at viewport `< 640px` (`sm` breakpoint), Display → 24px; pillar card score numeral → 36px. Body, Label, Heading sizes are unchanged across breakpoints.

**PDF typography**: governed by `src/lib/pdf/styles.ts`; do not introduce new sizes. The PDF mirror uses the existing PDF heading and body styles already shipped in Phase 5.

## Color

The 60 / 30 / 10 split for this phase:

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `#f8fafc` | `--color-background` | Report page background, modal scrim base |
| Secondary (30%) | `#ffffff` | `--color-surface` | Pillar card backgrounds, modal panel, admin form panels |
| Accent (10%) | `#1a365d` | `--color-navy` | Page heading, pillar card name, modal heading, primary CTA fill (e.g., admin "Save changes"), the active-state focus ring colour ramp |
| Brand highlight | `#F5A623` | `--color-gold` | Reserved-for list (see below) — NOT used as a generic accent in this phase |
| Destructive | `#ef4444` | `--color-rating-poor` | Admin "Delete prescription" confirmation; nothing in the client-facing report uses destructive colour |

**Navy (accent) reserved for:**
- Page heading "The Peak Living Pillars"
- Each pillar card's pillar name (heading text)
- Modal pillar name in modal header
- Primary admin CTA fill ("Save", "Save all pillars")
- Focus ring on interactive elements (uses existing `rgba(245, 166, 35, 0.25)` ring colour from `globals.css` — gold-tinted, kept consistent with the rest of the app)
- The collapsed disclosure summary label ("Detailed marker results")

**Gold (brand highlight) reserved for:**
- The Peak Living wordmark/word in the page intro paragraph if visually emphasised (one occurrence max)
- The "Updated by [Admin Name] · 2 days ago" attribution accent inside the modal prescription block (4px left border, gold)
- The brand identity strip at the very top of the report shell (existing pattern in Section 11) — unchanged
- NOTHING else. Gold is NOT used for traffic-light colours, NOT used as a generic CTA fill, NOT used on pillar cards.

**Traffic-light palette (pillar status — NEW, parallel to but distinct from the 5-tier marker palette):**

| Status | Fill | Text on fill | Border | Pending |
|--------|------|--------------|--------|---------|
| `green` (70–100) | `#10b981` | `#ffffff` | `#10b981` | n/a |
| `amber` (40–69) | `#f59e0b` | `#1a202c` | `#f59e0b` | n/a |
| `red` (0–39) | `#ef4444` | `#ffffff` | `#ef4444` | n/a |
| `pending` (no rated markers) | `#e2e8f0` (border) + `#64748b` (text) | n/a | `#e2e8f0` | dashed border, "Data pending" pill |

**These hex values live in a single source-of-truth module: `src/lib/pillars/colors.ts`.** Both the portal CSS-in-JS / Tailwind arbitrary values AND the PDF colour constants module import from this file (D-28). Do NOT duplicate the hex values in Tailwind config or PDF colour tables.

**Marker palette (5-tier) is preserved verbatim** inside the modal's "Score breakdown" section and inside the collapsed "Detailed marker results" disclosure. The 5 tier colours (`emerald / blue / gray / amber / red`) come from `TIER_COLORS` in `src/types/normative.ts` — already declared, not re-specified here.

**Anti-pattern call-outs** (prevent drift):
- Do NOT use the 5-tier `cautious` amber for the pillar `amber` status — they are different shades by design (`#f59e0b` is the same hex but its semantic meaning is "moderate / needs improvement at the pillar level", not "this single marker is in cautious tier"). Keep the visual usage distinct: 5-tier amber is a row-background; pillar amber is a foreground status fill.
- Do NOT use `red` (`#ef4444`) for the destructive admin action pattern AND a `poor` pillar status interchangeably — they coincide on the hex value but the destructive button is filled solid red with white text, while the pillar red badge is a small status pill or dot. The shape, not the colour, communicates the difference.

## Copywriting Contract

Phase 8 introduces user-facing copy across five surfaces: page hero, pillar cards, modal sections, admin authoring forms, and PDF mirror. Initial seed values for admin-editable copy live in the database migration; every other string below is hardcoded in the component layer.

### Page hero (admin-editable via `pillar_page_copy`)

| Element | Initial seed (locked from CONTEXT D-25) |
|---------|-----------------------------------------|
| Heading | **The Peak Living Pillars** |
| Intro body | Peak360 translates your results into five core pillars to show where you are performing strongly, where you may be exposed, and where focused intervention can help move you toward peak living. |

### Pillar definitions (admin-editable via `pillar_definitions`, seeded once)

| Pillar | Label seed | Short summary seed (1 line, shown on card under score) | Plain meaning seed (modal "What this pillar means") |
|--------|-----------|--------------------------------------------------------|------------------------------------------------------|
| `cardiometabolic` | Cardiometabolic Health | How well your heart, vessels, and metabolism are working together. | Cardiometabolic health covers the markers that drive long-term heart, vessel, and metabolic function — cholesterol balance, blood sugar control, inflammation, and blood pressure. Strong numbers here mean your engine is running clean. |
| `bodyComposition` | Body Composition | The balance of muscle, fat, and where it sits on your frame. | Body composition looks beyond weight — at the proportion of muscle to fat and how it's distributed. A favourable composition supports metabolic health, joint longevity, and how capable you feel day-to-day. |
| `strength` | Strength | Your ability to produce force — a leading indicator of healthy ageing. | Strength is one of the clearest signals of how well you are ageing. The ability to lift, push, and grip translates directly into independence, injury resistance, and quality of life as the decades roll on. |
| `balance` | Balance | Stability and proprioception — your body's quiet defence against falls. | Balance combines coordination, proprioception, and postural control. Strong balance now means a dramatically lower fall risk later, and it tracks closely with overall neurological and musculoskeletal health. |
| `vo2` | VO2 / Fitness Capacity | The size of your aerobic engine — the single best predictor of all-cause mortality. | VO2 max measures how efficiently your body uses oxygen during effort. It is one of the strongest predictors of healthspan and is highly trainable at any age. A larger aerobic engine means more capacity for everything else. |

**Anti-claims constraint (D-30):** every seed string above and any admin edit must avoid disease-prevention claims, longevity guarantees, and clinical diagnostic language. The admin authoring UI displays a one-line guidance note above the textarea: "Use plain English. Avoid medical claims, diagnoses, or longevity guarantees."

### Modal sections (locked order from CONTEXT specifics)

Section labels are hardcoded; body copy comes from data.

| Section | Label (Heading 20px) | Body source |
|---------|---------------------|-------------|
| 1. Header | (no label — pillar name + score + status pill stack) | `pillar_definitions.label`, computed score, computed status |
| 2. What this pillar means | What this pillar means | `pillar_definitions.plain_meaning` |
| 3. Your results in this pillar | Your results | Per-marker rows (5-tier coloured), filtered to the pillar's mapped categories |
| 4. What you are doing well | What you are doing well | Computed: markers in `great` or `elite` tier within this pillar, listed by `label` with their tier badge. Empty state copy: **No standout strengths in this pillar yet — every score below tells you where momentum can build.** |
| 5. What needs attention | What needs attention | Computed: markers in `cautious` or `poor` tier within this pillar. Empty state copy: **Nothing in this pillar is flagged for attention right now — keep it up.** |
| 6. Score breakdown | Score breakdown | Tabular: marker label, value + unit, tier badge, contribution to mean (`+20` … `+100`). Below the table, a single line: **Pillar score is the average tier value across {n} rated marker{s}. Markers without normative ranges are excluded.** |
| 7. Recommended plan | Recommended plan | `pillar_prescriptions.summary` (required), `bullets` (optional), `full_plan_href` CTA (optional, button label: **Open full plan**). ### Empty / loading / error states (across the report)

| State | Copy |
|-------|------|
| Pillar score `null` (no rated markers) | Card shows **—** in place of the number, dashed border, status pill **Data pending**. No traffic-light colour. |
| Modal "Your results" — pillar has zero rated markers | We don't have any rated markers for this pillar in this assessment yet. Check the detailed marker results below the pillars grid for any raw values your coach has entered. |
| Modal "Recommended plan" — no prescription | Your coach hasn't written a recommendation for this pillar yet. Check back soon. |
| Detailed marker results disclosure (collapsed by default) | Summary label (16px / 600): **Detailed marker results** · helper (14px / 400): For coaches and curious clients — every rated marker with raw values and ranges. |
| Server error loading the report | Couldn't load this report. Refresh the page, or contact your coach if the problem persists. |

### Primary CTA (admin surfaces)

| Surface | CTA |
|---------|-----|
| `/portal/admin/pillars` | **Save changes** (commits all five pillar definitions) |
| `/portal/admin/assessments/[id]/prescriptions` | **Save plan** (per pillar, recommended) — or **Save all pillars** (if a single form is preferred by the planner) |

The client-facing report has **no primary CTA** in this phase. The card itself is the affordance — tapping opens the modal. The modal's only CTA is the optional **Open full plan** link button when `full_plan_href` is set.

### Destructive actions

Only one destructive action ships in this phase:

- **Clear pillar prescription** (admin only, on `/portal/admin/assessments/[id]/prescriptions`).
  - Confirmation pattern: inline confirmation dialog (uses the same `Dialog` primitive as the pillar drill-down, in centred-dialog mode).
  - Confirm copy: see the table above.
  - Audit log entry: `pillar_prescription.delete` with `{ assessment_id, pillar_key, before_summary_hash }` payload (D-16).

No destructive actions exist on the client-facing report or the public-facing pillar cards.

## Layout & Responsive Contract

(Derived from CONTEXT D-22 / D-23 / D-25; this section makes the breakpoint behaviour explicit for the executor.)

| Surface | Mobile (< 768px) | Desktop (≥ 768px) |
|---------|------------------|-------------------|
| `PillarsGrid` | Single column, full-width cards, 16px gap, 16px page padding | 3-column grid for the first row (Cardiometabolic, VO2, Body Composition), 2-column grid for the second row (Strength, Balance) — matches D-22 sort order. 24px gap, max content width 1080px, centred. |
| `PillarCard` | min-height 96px, 16px padding, score numeral 36px | min-height 160px, 24px padding, score numeral 40px |
| `PillarModal` | Bottom-sheet: full-width, 90vh max-height, top corners rounded 16px, drag-handle pill on top, body scrolls inside, sticky close button top-right (44 × 44 hit target), backdrop scrim 50% black | Centred dialog: 640px max-width, 90vh max-height, 16px corner radius, 32px padding, body scrolls inside, close button top-right (44 × 44 hit target) |
| Detailed marker results disclosure | Collapsed by default, full-width, expands inline below the pillars grid | Same — collapsed by default, expansion does not change layout grid |
| `/portal/admin/pillars` | Single column form stack, one pillar per accordion row | Single column form stack, max-width 720px, centred — matches existing admin page shell (`/portal/admin/users`, `/portal/admin/invitations`) |
| `/portal/admin/assessments/[id]/prescriptions` | Single column form stack, one pillar per card | Single column form stack, max-width 720px, centred |
| PDF pillar page | n/a (fixed A4) | A4 portrait, 3 + 2 grid in a single page block, header strip + footnote |

**Focus & keyboard contract** (a11y, applies to portal modal):
- Tabbing into a pillar card: card is focusable (`tabindex=0`), `Enter` and `Space` open the modal.
- Modal opens: focus moves to the close button. `Esc` closes. `Tab` cycles inside the modal (focus trap). `Shift+Tab` cycles backward. Backdrop click closes.
- Modal closes: focus returns to the originating pillar card.
- All interactive elements have a visible focus ring (existing 2px gold-tint ring from `globals.css`).

**Reduced motion**: at `prefers-reduced-motion: reduce`, modal animation is replaced by an instant show/hide; bottom-sheet drag is unaffected. Pillar card hover transitions remain at 150ms (subtle, non-vestibular).
