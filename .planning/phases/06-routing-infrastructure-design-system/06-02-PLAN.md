  <name>Task 1: Configure DNS and DigitalOcean App Platform domains</name>
  <read_first>
    - .planning/phases/06-routing-infrastructure-design-system/06-RESEARCH.md (DNS pitfalls section, especially Pitfall 5 about apex domains)

  <what-built>Hostname-based middleware routing is implemented (Plan 01). The app rewrites apex domain requests to /landing and serves portal on the subdomain. DNS needs to be configured so both domains point to the DigitalOcean app.</what-built>
  <how-to-verify>
    Configure the following:

    **Step 1: Add domains in DigitalOcean App Platform**
    1. Go to DigitalOcean dashboard -> Apps -> your Peak360 app -> Settings -> Domains
    2. Add `peak360.com.au` as a custom domain
    3. Add `portal.peak360.com.au` as a custom domain
    4. Note the CNAME target DigitalOcean provides (usually something like `your-app.ondigitalocean.app`)

    **Step 2: Configure DNS records**
    At your DNS provider for peak360.com.au:
    - For `portal.peak360.com.au`: Add a CNAME record pointing to the DigitalOcean target
    - For `peak360.com.au` (apex): Either use ALIAS/ANAME record (if provider supports it, e.g., Cloudflare CNAME flattening) OR migrate DNS to DigitalOcean which handles apex domains natively

    **Step 3: Verify SSL**
    Wait for DigitalOcean to auto-provision SSL certificates (usually 5-15 minutes).

    **Step 4: Test**
    - `curl -I https://peak360.com.au` should return 200
    - `curl -I https://portal.peak360.com.au` should return 200
    - Both should show valid SSL certificates
  </how-to-verify>
  <resume-signal>Type "dns-configured" when both domains resolve correctly, or describe any issues encountered</resume-signal>
  <acceptance_criteria>
    - `dig peak360.com.au` returns a valid A or CNAME record
    - `dig portal.peak360.com.au` returns a valid CNAME record
    - `curl -sI https://peak360.com.au | head -1` shows HTTP 200 or 301
    - `curl -sI https://portal.peak360.com.au | head -1` shows HTTP 200 or 301
  </acceptance_criteria>
  <done>Both peak360.com.au and portal.peak360.com.au resolve to the DigitalOcean app with valid SSL. Hostname-based routing serves the correct content on each domain.</done>
