    1. Parse hostname from `req.headers.get('host')` stripping port number: `const hostname = req.headers.get('host')?.split(':')[0] ?? '';`

    2. Define landing hostnames from env var with defaults:
       ```typescript
       const LANDING_HOSTNAMES = new Set(
         (process.env.LANDING_HOSTNAMES ?? 'peak360.com.au,www.peak360.com.au').split(',')
       );
       ```

    3. After the static asset passthrough (existing `/_next`, `/favicon`, `.png`, `.svg`, `.ico` checks), add hostname routing BEFORE the auth logic:

       ```typescript
       // Hostname routing: landing page vs portal
       if (LANDING_HOSTNAMES.has(hostname)) {
         // Root path on landing domain -> rewrite to /landing route
         if (pathname === '/') {
           const url = req.nextUrl.clone();
           url.pathname = '/landing';
           return NextResponse.rewrite(url);
         }
         // API routes on landing domain pass through (e.g., contact form in Phase 7)
         if (pathname.startsWith('/api/')) {
           return NextResponse.next();
         }
         // All other paths on landing domain -> redirect to portal
         const portalUrl = new URL(pathname, 'https://portal.peak360.com.au');
         return NextResponse.redirect(portalUrl);
       }

       // Block direct access to /landing on portal hostname
       if (pathname === '/landing' || pathname.startsWith('/landing/')) {
         return NextResponse.redirect(new URL('/', req.url));
       }
       ```

    4. Keep ALL existing auth logic unchanged below the hostname routing block. The auth checks (ADMIN_PASSWORD, session cookie, validateSessionToken) remain exactly as they are.

    5. Add `/landing` to the PUBLIC_PATHS set so that rewrites to /landing don't trigger auth on landing hostname. Actually, the rewrite happens before auth so landing requests return before reaching auth. But direct /landing access is redirected away. No change to PUBLIC_PATHS needed.

    The full middleware flow should be:
    - Static assets -> pass through
    - Landing hostname + root -> rewrite to /landing
    - Landing hostname + /api -> pass through
    - Landing hostname + other -> redirect to portal
    - Portal hostname + /landing -> redirect to /
    - Public paths (/login, /api/auth/*) -> pass through
    - Auth checks (existing logic unchanged)
