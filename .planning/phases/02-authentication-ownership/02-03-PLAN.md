   - Import `auth` from `@/lib/auth`, `headers` from `next/headers`, `sendEmailViaSMTP2Go` from `@/lib/email/send`
   - Import `db` and user table from schema
   - Export async POST handler:
     a. Validate session: `const session = await auth.api.getSession({ headers: await headers() })`. Return 401 if no session.
     b. Check role: only `coach` or `admin` can invite. Return 403 if `session.user.role === 'client'`.
     c. Parse body: `{ email: string }`. Return 400 if email missing or invalid.
     d. Check if user already exists with this email in the `user` table. If exists and role is already `client`, return `{ success: true, message: "Client already exists, sent login link" }` and send a magic link email instead.
     e. If user does not exist: use Better Auth's server API to create the user with role `client`. Call `auth.api.signUpEmail({ body: { email, password: crypto.randomUUID(), name: email.split('@')[0] } })` then update role to `client` in DB.
     f. Send invite email via `sendEmailViaSMTP2Go`:
        - to: the client email
        - subject: "You've been invited to Peak360"
        - html: A simple HTML email with: "Your coach has invited you to view your health assessments on Peak360. Click the link below to sign in:" followed by a magic link URL. Use `${process.env.BETTER_AUTH_URL ?? 'http://localhost:3000'}/login?mode=client` as the link (client will request magic link from login page).
     g. Return `{ success: true, message: "Invitation sent" }`

2. Add a minimal invite UI to the coach dashboard (`src/app/(portal)/portal/page.tsx`):
   - Add a small "Invite Client" section — an inline form with:
     - An email input field (type="email", placeholder="Client email address")
     - A "Send Invite" button
   - On submit: POST to `/api/invitations` with `{ email }`. Show success/error message below the form.
   - Style with existing Tailwind classes: `flex gap-2` layout, input uses `px-3 py-2 border border-border rounded-md text-sm`, button uses `bg-gold text-navy px-4 py-2 rounded-md font-medium hover:bg-gold/90`
   - Show loading state on button ("Sending...") and disable during request
   - This section should only be visible if the logged-in user is a coach or admin (use `authClient.useSession()` to check role)
  </action>
