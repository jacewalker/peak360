    Create `src/lib/crypto.ts` implementing AES-256-GCM encryption per D-05 and D-06:

    1. `getKey(): Buffer | null` -- reads `process.env.ENCRYPTION_KEY`, returns null if unset. Supports hex (64 chars) and base64 (44 chars) encoded 32-byte keys.
    2. `encrypt(plaintext: string): string` -- generates random 12-byte IV, encrypts with AES-256-GCM, returns `base64(IV + authTag + ciphertext)`. Returns plaintext unchanged if no key (D-08).
    3. `decrypt(ciphertext: string): string` -- extracts IV (first 12 bytes), authTag (next 16 bytes), ciphertext (remainder) from base64 decoded buffer. Returns plaintext unchanged if no key or if `isEncrypted()` returns false (backwards compat).
    4. `isEncrypted(value: string): boolean` -- returns false if value starts with `{` or `[` (JSON). Otherwise base64-decodes and checks buffer length >= 29 (12 IV + 16 tag + 1 data).

    Use Node.js built-in `crypto` module only -- `createCipheriv`, `createDecipheriv`, `randomBytes`. No external dependencies.

    Constants: `ALGO = 'aes-256-gcm'`, `IV_LENGTH = 12`, `TAG_LENGTH = 16`.

    Create `src/lib/crypto.test.ts` with vitest tests covering all behaviors above. Set `process.env.ENCRYPTION_KEY` to a test hex key (64 hex chars) in beforeEach, delete it in specific "unset" tests.
